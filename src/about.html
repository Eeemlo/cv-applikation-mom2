Om uppgiften

Uppgiften bestod av två delar där den första delen gick ut på att skapa en webbtjänst med NodeJs, Express samt en valfri relationsdatabas. Jag valde att använda mig av PostgreSQL för att kunna publicera webbtjänsten via Render. Ett krav för webbtjänsten var att den ska kunna hantera CRUD-operationerna CREATE, READ, UPDATE och DELETE samt tillåta Cross Origin Requests.


Den andra delen gick ut på att skapa en webbapplikation som konsumerar den webbtjänst jag skapat i tidigare moment. På webbplatsen används Fetch API för att hämta, skicka samt radera data från och till webbtjänsten.

Slutsatser

Jag tyckte att det var relativt enkelt att sätta upp databasen och servern som hanterar mitt API, då vi nyligen skrivit liknande kod i tidigare moment. Däremot upplevde jag det desto svårare att skriva JavaScript-kod med fetch API för att skapa och radera poster i mitt API, då detta är något vi aldrig tidigare gjort. Det kommer troligtvis att ta ett tag innan jag fullt ut förstår logiken bakom koden, men det har varit oerhört lärorikt att få testa på att skapa sitt eget API och utföra CRUD-operationer till det. Nu behöver jag bara öva lite till!
